##### 사용 라이브러리:

Win32API - 게임 화면을 출력해줄 라이브러리, MFC보다 저수준 라이브러리로 DirectX의 출력만 해주면 되는 윈도우 프로그램이므로 더 가벼운 WIn32API를 사용

DirectX11 - 고성능 랜더링을 위한 라이브러리

MFC - tool용, 실제 게임을 위한 프로그램이 아니므로 FPS가 중요하지 않아 여러 편의성을 제공해줄 요소로 MFC로 툴을 만듬.



##### 프로젝트 폴더 구조 설명:

###### DXEngine

* 게임 개발의 기본 기능과 핵심 시스템을 제공하는 엔진 부분.
* 렌더링, 리소스 관리, 입출력 등 개발 전반에 필요한 기반 기능 포함.

###### DXMiddle

* Client와 Tool 모두에서 공통으로 사용하는 라이브러리와 기능을 통합한 중간 계층.
* 코드 재사용과 모듈화를 위해 설계됨.

###### DXClient

* 최종 사용자에게 보여질 게임 프로그램 부분.
* 게임 콘텐츠와 Client 전용 기능이 포함되어 있으며, 실제 플레이 가능한 결과물 담당.

###### DXTool

* 개발 편의성을 위한 툴 전용 코드.
* MFC 기반 UI 처리, 툴 인터페이스, 콘텐츠 편집 기능 등을 포함.
